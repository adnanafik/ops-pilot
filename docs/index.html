How it works

ops-pilot runs four agents in sequence: Monitor → Triage → Fix → Notify.

MonitorAgent polls GitHub Actions for failed runs and builds a typed Failure model from the log tail.

TriageAgent sends the logs to Claude and extracts root cause, severity, and fix confidence into a structured Triage model.

FixAgent asks Claude which file to edit, fetches it from GitHub, generates a minimal patch, and opens a draft PR — humans review before anything merges.

NotifyAgent writes a concise Slack message via Claude and posts it to the configured channel.

This demo replays pre-recorded runs — click any scenario above to see the full pipeline in action.
